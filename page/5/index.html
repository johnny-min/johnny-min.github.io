celery 简介

  1. web里的发邮件就可以使用它，当点击发邮件后主程序继续运行而不必等待邮件过程。

    celery 架构

  3. Celery Baet: 任务调度器，读取配置文件内容，将任务发送给队列；
  4. Celery Worker: 执行任务的消费者；
  5. Broker: 消息代理（消息中间件），接受生产者发过来的任务消息，存进队列再按顺序分发给消费方；
  6. Producer：生产者；
  7. Result Backend：任务处理完后保存状态信息和结果。

    消息代理

hexo + github page 搭建博客


**什么是hexo?**
- Hexo 是一个快速、简洁且高效的博客框架。其使用Markdown解析文章。
**搭建步骤(WIN 10 为例)**
- 1.安装Node.js
可以在：https://nodejs.org/zh-cn/download/ 这个链接中下载安装包（安装包可以自动安装依赖和配置环境），安装完成后，可以在命令行中输入 node -v 来检测是否安装成功。
- 2.安装Hexo
通过npm(软件包管理工具)命令来安装： npm install -g hexo-cli,安装完成后输入 hexo -v 来验证是否安装成功。
- 3.hexo命令
hexo init blog 初始化文件夹（必须是空文件夹） hexo g 生成静态网页 hexo s 打开本地服务器
- 4.github设置
在github上新建一个项目(New repository),然后在Respository name 中输入项目名，后缀必须是.github.io,名称和github名一样。
- 5.连接Github和本地
git config --global user.name 'test' git config --global user.email 'test@gmail.com'
- 6.blog配置：
打开博客根目录下的 _config.yml(博客的配置文件),在最后一行加入： 
deploy:
  type: git
  repository: https://github.com/username/username.github.io
  branch: master

- 7.输入命令
hexo clean
hexo g
hexo d
- 8.发布文章
输入 hexo n 'blogname'会发现在/source/_posts目录下多出来个 blogname.md 文件 然后再markdown中编辑即可。可以使用 hexo s --debug 在本地浏览效果(localhost:4000),也可在markdoown中浏览效果，然后通过hexo g hexo d 生成和部署
